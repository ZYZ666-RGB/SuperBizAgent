# SuperBizAgent Evaluation Runbook

## Redis Timeout

When order-service has Redis timeout, the first response should check Redis connection pool saturation,
timeout settings, Redis node health, network latency, and the retry budget. Preserve error logs before
restarting the service.

## High CPU

When payment-service has high CPU usage, inspect hot threads, recent deployments, garbage collection,
container CPU limits, and incoming request rate. Scale out before risky restarts.

## Memory OOM

When a JVM service has OOM or high memory usage, check heap usage, Full GC frequency, heap dump evidence,
memory leak patterns, and recent traffic changes.
